Complet Atelier : Générer et utiliser une signature numérique

Objectifs

=  Utilisez OpenSSL pour générer une signature numérique.

=  Signer un document avec la signature numérique.

=  Vérifier qu'un document signé a été modifié.

Contexte/Scénario

Une signature numérique est une technique mathématique utilisée pour valider l'authenticité et l'intégrité d'un message numérique. L'objectif d'une signature numérique est d'empêcher la falsification et l'usurpation d'identité dans les communications numériques. Dans de nombreux pays, comme aux États-Unis, les signatures numériques ont le même poids juridique que les signatures traditionnellement apposées aux documents papier. Il arrive même que le gouvernement américain publie des versions électroniques de ses budgets, lois et projets de loi dotées de signatures numériques.

Un algorithme de signature numérique consiste en un processus de création et de vérification de la signature. L'utilisateur A génère la signature numérique et l'utilisateur B vérifie la signature à l'aide du processus de vérification. Le signataire et le vérificateur ont tous deux une clé publique et une clé privée qu'ils utilisent pour terminer chaque processus.

Dans ce TP, vous allez utiliser la boîte à outils OpenSSL pour générer une signature numérique. Vous générerez ensuite un document, le signerez avec la signature numérique, puis validerez l'authenticité et l'intégrité du document. Enfin, vous modifiez le document, puis vous confirmez que le document n'est plus authentique, car son intégrité a été compromise.

cisco@labvm:~$ openssl pkey -in private_key.pem -pubout -out public_key.pem

Partie 4 : Créez un nouveau document qui sera signé numériquement.

a.  Utilisez la commande echo pour créer un fichier texte nommé contract.txt.

cisco@labvm:~$ echo Please transfer 2,000,000 US Dollars to Mr. Jester by 6pm today! > contract.txt

cisco@labvm:~$  

b.  Utilisez la commande cat pour afficher le fichier contract.txt.

cisco@labvm:~$ cat contract.txt

Please transfer 2,000,000 US Dollars to Mr. Jester by 6pm today!

Partie 5 : Utilisez la clé privée pour signer numériquement le nouveau document.

a.  Pour signer le document, utilisez la commande openssl dgst. La commande dgst peut prendre un nombre illimité de valeurs de résumé de message. Dans cet exemple, vous allez utiliser SHA 256, puis private_key.pem pour générer une signature pour le document contract.txt.

cisco@labvm:~$ openssl dgst -sha256 -sign private_key.pem -out signature contract.txt

cisco@labvm:~$

Partie 6 : Vérifier l'authenticité et l'intégrité du document.

La technologie de signature numérique permet au destinataire de vérifier l'authenticité et l'intégrité du fichier. Le processus de vérification de la signature numérique consiste à s'assurer qu'un message donné a été signé par la clé privée qui correspond à une clé publique donnée.

Pour vérifier que le document est authentique et qu'il n'a pas été falsifié, utilisez la commande openssl dgst avec l'option de verify (vérification) et le fichier public_key.pem.

cisco@labvm:~$ openssl dgst -sha256 -verify public_key.pem -signature signature contract.txt

Verified OK

Partie 7 : Simulez un hacker qui change le destinataire spécifié dans le fichier contract.txt.

a.  Utilisez gedit pour ouvrir le fichier contract.txt.

cisco@labvm:~$ gedit contract.txt

b.  Remplacez Mr. Jester par Mr. Viper.

c.  Cliquez sur File (Fichier) > Quit (Quitter), puis sur Save (Enregistrer) dans la boîte de dialogue.

Partie 8 : Vérifiez que l'intégrité du document a été compromise.

Réutiliser la commande openssl dgst avec l'option verify pour valider que la vérification du document échoue maintenant.

cisco@labvm:~$ openssl dgst -sha256 -verify public_key.pem -signature signature contract.txt

Verification Failure

cisco@labvm:~$
